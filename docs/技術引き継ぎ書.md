## 3. スプレッドシート構造（列順は絶対に変えない：GASが列番号参照）

- **商品マスタ**：A商品ID / B商品名 / Cカテゴリ / D状態 / E発注起点 / F発注窓口 / G実製造元 / H原料の渡し方 / I納品先 / Jリードタイム日 / K完成報告 / L備考
  - 商品ID＝`P01`〜（GASが `P(\d+)` の最大値+1で自動採番、2桁ゼロ埋め）
  - 状態＝`販売中` or `終売`
- **入出庫ログ**：AログID / B日付 / C商品ID / D商品名 / E区分（入庫/出庫/調整）/ F数量 / GロットNo / H出庫期限 / I重量kg / J備考 / K登録者 / L登録日時
  - 在庫＝Σ入庫−Σ出庫＋Σ調整（調整は±の符号付き数量）
  - 移行データ4,151件（登録者=`旧表移行`）。移行時調整ログあり
- **発注管理**：A発注ID / B商品ID / C商品名 / D発注日 / E数量 / F原料指定日 / Gステータス / H備考 / I更新日時
- **設定**：Aキー / B値（`安全在庫月数`=0.5、`SLACK_WEBHOOK_URL`=空なら通知無効）
- **要確認**：変換時に解析不能だった旧データの一覧（運用には影響しない）

## 4. GAS API仕様（gas-code.gs）

- `doGet` → 全状態JSON。`doPost`（body=JSON文字列、Content-Type: text/plain）→ アクション実行後に**全状態JSONを返す**（クライアントはそれで画面更新＝1往復）
- レスポンス形：`{ok:true, master:[{pid,name,cat,status,trigger,vendor,maker,material,dest,lt,report,memo}], settings:{}, stock:{pid:数}, monthly:{pid:{"yyyy-MM":数}}(直近14か月・出庫のみ), recentLogs:[直近25件], orders:[進行中全件+完了直近30件], serverTime:"yyyy-MM-dd HH:mm"}`
- アクション：`addLog{type,pid,pname,date,qty,lot,expiry,memo,user}` / `addOrder{pid,pname,date,qty,mat,memo}` / `updateOrder{oid,status|mat|qty|memo}` / `deleteOrder{oid}` / `addProduct{product}` / `updateProduct{pid,product}` / `deleteProduct{pid}`
- 書き込みは LockService で直列化。Slack通知は addLog / addOrder 時、Webhook設定時のみ
- **再デプロイ時の注意**：「デプロイを管理」→既存デプロイを新バージョンで更新（新規デプロイするとURLが変わりアプリ側の書き換えが必要）

## 5. アプリ仕様（pb-manager.html）

- 冒頭 `const GAS_URL = "";` にデプロイURLを貼ると**オンラインモード**。空なら**ローカル試用モード**（localStorage保存・SEED_MASTER使用）。オンライン時は最終状態を`pbcache`にキャッシュしオフライン閲覧可
- タブ：ホーム（KPI・発注アラート・終売残在庫・進行中案件）/ 発注案件 / 予測・発注点 / 入出庫 / マスタ
- **発注点ロジック**：`発注点=直近3か月平均販売×(LT日/30＋安全在庫月数)`。在庫≦発注点→`★要発注`、在庫残月数≦LT月+安全+0.5→`まもなく`。実績3か月未満は`実績不足`
- **予測**：直近3か月平均＋直近6か月の単回帰傾き（実績4か月以上のときのみ傾き適用）で3か月先まで
- **終売仕様**：発注アラート・発注登録・入庫の対象外。在庫>0の間は出庫可能。マスタで販売中⇄終売をトグル可
- **工程**：発注済→原料手配中→原料発送・引渡済→製造中→完成報告受領→入庫依頼済→納品完了（「次の工程へ▶/◀戻す」ボタン式）
- 出庫時に在庫超過なら confirm 警告。登録者名は localStorage `pbuser` に記憶
- POSTは `Content-Type: text/plain` 固定（CORSプリフライト回避のため。**application/jsonに変えないこと**）

## 6. 重要な決定事項（経緯）

1. 旧Excelの「豚まん(旧)」シートは「豚まん」と重複データのため変換除外
2. 移行時、旧表の入庫残を正として「調整」ログで一致させた。**例外：P03 GPベーコンクリーム**は旧表が異常値(45,966)のため未調整→**運用開始前にヨコレイ在庫証明で棚卸し、アプリの棚卸調整で確定する（未完了タスク）**
3. 終売品：P15 GPハムナゲット(残34)・P16 GPロースハムコロッケ(残144)は売切り管理。P17 MMまん・P18 GPピザ・P19 ハンバーグ(業務用)は在庫0の取扱終了
4. P14 GP角煮は発注フロー未把握のため窓口等が「（要確認）」のまま登録（ユーザー本人も把握していない商品）
5. LTは変動幅がある商品は最大値を採用（メンチ/豚まん/焼売=30日、コロッケ/ベーコンクリーム=45日）
