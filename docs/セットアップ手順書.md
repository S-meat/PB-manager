# PB商品 受発注管理システム セットアップ手順書

**目標：7月7日（火）運用開始**　所要時間：約60分（＋棚卸確認）

## 全体構成

```
[PB管理アプリ]  ←GAS Web API→  [Googleスプレッドシート]
GitHub Pages                     商品マスタ / 入出庫ログ / 発注管理 / 設定
（本店精肉・製造部・営業部）       （データの正本・涼さんのGoogleアカウントで管理）
                                        │
                                        └→ Slack自動通知（任意）
```

- **アプリ**＝全員の入口。出庫・入庫・発注・マスタ管理はすべてここから
- **スプレッドシート**＝データの正本。直接編集は原則しない（閲覧・監査用）
- 旧「ヨコレイ在庫管理表.xlsx」は運用開始後、**閲覧専用**にして凍結

---

## STEP 1：スプレッドシートの作成（10分）

1. `ヨコレイ在庫_スプレッドシート投入用.xlsx` を Google ドライブへアップロード
2. 開いて「ファイル」→「**Googleスプレッドシートとして保存**」
   ※必ず変換すること。xlsxのままではGASが使えません
3. 変換後のファイル名を「PB商品受発注管理DB」等に変更
4. シート構成を確認：`商品マスタ`（19商品）／`入出庫ログ`（4,151件の移行データ）／`発注管理`（空）／`設定`／`要確認`

> 投入用ファイルには2017年からの全入出庫履歴と、旧表の入庫残に合わせる「移行時調整」ログが含まれています。

## STEP 2：GASの設置とデプロイ（15分）

1. 変換したスプレッドシートで「拡張機能」→「**Apps Script**」
2. 既存コードを消して `gas-code.gs` の内容を全文貼り付け → 保存
3. 動作確認：関数の選択で `testBuildState` を選んで実行
   - 初回は権限の承認画面が出ます →「許可」
   - 実行ログにJSONが出ればOK
4. 「デプロイ」→「新しいデプロイ」→ 種類「**ウェブアプリ**」
   - 説明：任意（例：v1）
   - 実行ユーザー：**自分**
   - アクセスできるユーザー：**全員**
5. 発行された **ウェブアプリURL**（`https://script.google.com/macros/s/…/exec`）をコピー

> ⚠️ コード修正時は「デプロイを管理」→ 既存デプロイの編集 → 新バージョンで更新すること（新規デプロイにするとURLが変わります）

## STEP 3：アプリの接続とGitHub Pages公開（15分）

1. `pb-manager.html` をエディタで開き、上部の
   `const GAS_URL = "";` に STEP 2 のURLを貼り付けて保存
2. GitHub（s-meat）に新規リポジトリ `pb-manager` を作成し、`index.html` にリネームしてアップロード
3. Settings → Pages → Branch: main で公開
4. `https://s-meat.github.io/pb-manager/` を開き、ヘッダーが
   「**スプレッドシート連携中（最終同期 …）**」になっていれば接続成功

## STEP 4：Slack通知の設定（任意・5分）

1. Slackで Incoming Webhook を作成（通知先：`#201-特注共有` など）
2. スプレッドシートの `設定` シート、`SLACK_WEBHOOK_URL` の値欄にWebhook URLを貼る
3. 以後、出庫・入庫・調整・発注登録のたびに自動通知（現在庫付き）

## STEP 5：切替時の棚卸確認（運用開始前に必須）

- **GPベーコンクリーム（P03）**：旧表の入庫残が異常値（45,966）だったため、移行時調整をしていません。**ヨコレイの在庫証明等で実在庫を確認**し、アプリの「入出庫」→「棚卸調整」で差分を登録して確定させてください
- 他の商品は旧表の入庫残に合わせ済みですが、可能ならヨコレイ月報と突き合わせて開始在庫を確認することを推奨します
- スプレッドシートの `要確認` シートに、変換時に解析できなかった旧データの一覧があります（過去分のため運用には影響しません）

## STEP 6：運用ルールの周知

| 誰が | いつ | 何を |
|---|---|---|
| 本店精肉 | 出庫依頼のとき | アプリ「入出庫」→ **出庫** を登録（登録＝依頼の記録。Slackで製造部へ自動通知） |
| 製造部 | 横浜冷凍等へ納品したとき | アプリ「入出庫」→ **入庫** を登録（ロットNo・出庫期限も入力推奨） |
| 発注担当（→将来は営業部） | ホームの「★要発注」が出たとき | 「発注を登録」→ 以後は工程を「次の工程へ」で進める |
| 発注担当 | FAX・メール受領のたび | 該当案件の工程を更新（原料指定日も入力） |
| 管理者 | 商品の追加・終売時 | 「マスタ」タブから追加／終売／編集 |

- 在庫・販売実績・発注点はすべて自動計算。手入力は入出庫と発注だけ
- 旧Excelへの記入は7月7日以降**停止**（二重管理禁止）

## トラブルシューティング

- **「オフライン」表示**：通信環境を確認して「更新」。前回同期時点のデータは閲覧可能
- **送信に失敗する**：GASのデプロイ設定が「全員」になっているか確認
- **マスタを直したい**：アプリの「マスタ」タブから。スプレッドシート直編集も可能だが、列の順序は変えないこと
- **データの控え**：アプリの「控え保存」でJSONバックアップ。スプレッドシート自体もGoogleの版履歴で復元可能

## 営業部移管（Phase 2）に向けて

このシステムの管理者権限（スプレッドシートのオーナー・GAS）を営業部担当者へ共有すれば、移管はURL共有だけで完了します。提案書のPhase 2協議の際にこの手順書を添付してください。
